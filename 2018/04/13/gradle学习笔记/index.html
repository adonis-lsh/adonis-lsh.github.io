定义

gradle 命令会在当前目录中查找一个叫 build.gradle 的文件,它是一个构建配置脚本 (build configuration script). 这个脚本定义了一个 project 和它的 tasks. “<<” 表示向 helloWorld 中加入执行代码,创建hello之后执行,等同于doLast.

task hello {
    doLast {
        println 'Hello world'
    }
}
task helloWorld << {
    println "Hello World"
}

执行代码,在相同的目录,gradle -q helloWorld,-q代表 quiet 模式. 它不会生成 Gradle 的日志信息 (log messages).

语法(见一个学一个)

  1. 兼容java语法
  2. times,整数循环的终点,循环还有其他upto
4.times {print("$it")}

任务依赖

依赖

task taskB << {
    println("我是任务B")
}
task taskA(dependsOn: taskB) << {
    println("我是任务A")
}

在taskA执行前会执行taskB,但是前提是taskB必须提前定义

懒依赖

task taskX(dependsOn: 'taskY') << {
    println("我是任务X")
}
task taskY << {
    println("我是任务Y")
}

后面加了’’号的taskY,相当于先进行了声明,所以taskY可以在去用的时候在取值

动态任务

4.times {i ->
    task "task$i" << {
        println("I'm task number $i")
    }
}

运用了lambda表达式,挺好

> gradle -q task0
I'm task number 2
I'm task number 3
I'm task number 0
task0.dependsOn task2, task3

task hello << {
    println("Hello Earth")
}

hello.doLast{
    println("任务之后")
}

hello.doFirst{
    println("任务之前")
}

hello << {
    println("I am lastest")
}

当任务执行的时候, 在动作列表里的动作将被按顺序执行. 这里第四个行为中 << 操作符是 doLast 的简单别称.看一眼全明白了,soeasy

task hello << {
    println('Hello world')
}

hello.doLast {
    println("来自$hello.name task 的问候")
}

task 可以看做gradle的属性,所以可以用$任务名表示

自定义任务属性

task myTask{
    ext.myProperty = "myValue"
}

task printTaskProps << {
    println(myTask.myProperty)
}

note ext是extensions的缩写,可以直接写成extensions

调用Ant任务

** Ant任务 ** ant 是一个将软件编译、测试、部署等步骤联系在一起加以自动化的一个工具，大多用于Java环境中的软件开发。在实际软件开发中，有很多地方可以用到ant。Ant运行是需要一个xml文件

task loadFile << {
    def files = file('xml').listFiles().sort()
    files.each { File file ->
        if (file.isFile()) {
            ant.loadfile(srcFile: file,property: file.name)
            println("**$file.name")
            println("${ant.properties[file.name]}")
        }
    }
}

def 用来定义类型,each是遍历,ant.properties[file.name]是取所有属性中的file.name的属性 ant.loadfile(srcFile: file,property: file.name) 其实配置的就是下面哪个xml文件的配置

<loadfile property="message"
          srcFile="message.txt"/>

** 抽取封装 **

File[] fileList(String dir) {
    file(dir).listFiles({file -> file.isFile()} as FileFilter).sort()
}

task loadFile << {
    fileList("xml").each {File file ->
        ant.loadfile(srcFile: file,property: file.name)
        println("I'm found of $file.name")
    }
}

这种脚本语言写着就是爽,真是语法糖可以甜死我,尤其是as的使用,把这一点体现的淋漓尽致

默认任务

defaultTasks 'clean','run'

task clean << {
    println("默认清理任务")
}

task run << {
    println("默认运行任务")
}

task other << {
    println("其他任务")
}
> gradle -q

相当于执行 gradle -q clean run

DAG配置

task distribution << {
    println("version=$version")
}

task debug(dependsOn: distribution) << {
    println("正在编译debug")
}

task release(dependsOn: distribution) << {
    println("正在编译release")
}


gradle.taskGraph.whenReady { taskGraph ->
    if (taskGraph.hasTask(debug)) {
        version = 'debug 1.0'
    } else {
        version = "release 1.0"
    }
}

这里面的version本身就是就是已有的字段,已经gradle可以直接使用,所以没有定义的字段.其中taskGraph.whenReady在graph has been populated的时候执行,相当于监听.API文档
